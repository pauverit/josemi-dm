CONTEXTO Y DECISION CLAVE
================================================================================
Este proyecto implementa las funcionalidades del roadmap "Neutral SaaS" pero
adaptadas EXCLUSIVAMENTE para Digital Market. NO hay tenants. NO hay tenant_id.
Es un ERP privado para una sola empresa.

Necesidades específicas de Digital Market:
  - 2 CIFs (dos entidades fiscales / sociedades)
  - 4 delegaciones (sucursales)
  - 4 almacenes
  - Usuarios ilimitados con los roles que haya

Objetivo: Presupuesto → Pedido de venta → Albarán → Factura (order-to-cash)

        - Conversión a factura con un clic

  2.4 Tabla `facturas` + `factura_lineas`
        - Numeración por serie (A-0001 para CIF1, B-0001 para CIF2...)
        - Vinculada a pedido/albarán
        - Registro de cobro con fecha y método
        - VeriFactu: hash + encadenamiento (básico)
        - PDF legal imprimible

  2.5 Adaptar el flujo existente de pedidos B2B para que genere pedidos_venta

Objetivo: Gestión completa de stock por almacén, PMP, traspasos entre almacenes

Tareas:

  3.6 Tabla `traspasos` + `traspaso_lineas`
        - Origen almacén, destino almacén
        - Estado: en_transito → recibido
        - Firma digital obligatoria para confirmar recepción
        - Albarán de traslado imprimible

  3.7 Landed Costs (costes de importación)
        - Al recibir OC: añadir flete, aranceles, seguro
        - Reparto por valor/peso/unidades
        - Recalcula PMP sumando estos costes

  3.8 UI: panel de stock por almacén, movimientos, traspasos

NOTAS TÉCNICAS
================================================================================

Sin tenants:
  - NO usar tenant_id en ninguna tabla nueva
  - La empresa se identifica por empresa_id (tabla empresas)
  - Las políticas RLS de Supabase son por rol de usuario, no por tenant

Roles y permisos:
  - superadmin / admin → acceso total
  - direccion         → lectura de todo, reportes
  - administracion    → facturas, contabilidad, RRHH, SEPA
  - sales_lead        → CRM completo, presupuestos, pedidos
  - sales             → CRM propio, portal B2B
  - compras           → módulo compras, proveedores, OC
  - almacen           → stock, traspasos, recepciones
  - tech_lead         → SAT completo + agenda técnicos
  - tech              → SAT propio, partes de trabajo
  - client            → portal B2B, facturas propias, estado SAT

Ficheros clave del proyecto:
  - App.tsx           → router principal de vistas (~1700 líneas)
  - types.ts          → tipos TypeScript de todo el dominio
  - constants.ts      → constantes (comerciales, productos iniciales)
  - lib/supabase.ts   → cliente Supabase
  - components/       → todos los componentes React
  - hooks/            → custom hooks de datos
  - services/         → orderService.ts

SQL migrations creadas:
  - sat_schema.sql
